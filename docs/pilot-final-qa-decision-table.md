# 파일럿 전 최종 QA 판정표

이 문서는 **리더십 딜레마 365: 위기의 12주** 파일럿 운영 직전에 사용하는 최종 판정표입니다.

목표는 운영자가 감으로 판단하지 않고, 문서·화면·CI·리허설 결과를 기준으로 **운영 가능 / 조건부 가능 / 보류**를 결정하는 것입니다.

---

## 1. 최종 판정 기준

| 판정 | 의미 | 운영 판단 |
|---|---|---|
| 운영 가능 | 핵심 동선과 리포트가 정상 작동 | 예정대로 파일럿 진행 |
| 조건부 가능 | 일부 보완점은 있으나 교육 진행은 가능 | 강사 안내와 백업 절차를 준비하고 진행 |
| 보류 | 핵심 동선 또는 데이터가 불안정 | 원인 수정 후 재점검 |

---

## 2. 필수 QA 항목

| 영역 | 확인 항목 | 운영 가능 | 조건부 가능 | 보류 |
|---|---|---|---|---|
| 빌드 | `npm run build` | 성공 | 로컬 실패지만 Vercel 배포 성공 | 로컬/배포 모두 실패 |
| CI | GitHub Actions 또는 배포 check | 성공 | 실행 이력 없음, 수동 확인 완료 | 실패 원인 미해결 |
| 방 생성 | `/host/create` | 방 생성과 입장 코드 정상 | 새로고침 후 재시도 필요 | 방 생성 불가 |
| 참가자 입장 | `/join/:joinCode` | 참가자 등록 정상 | 일부 브라우저 안내 필요 | 참가자 등록 불가 |
| Round 0 | KSA 선택 | 팀별 KSA 저장 정상 | 일부 팀 수동 재확인 필요 | 역량 프로필 생성 불가 |
| 플레이 라운드 | Week 1~11 | 팀 결정과 산출물 저장 정상 | 일부 라운드 강사 보조 필요 | 결정/산출물 저장 불가 |
| 결과 계산 | 결과 카드/상태값 | 결과 카드 정상 | 결과 문구 일부 보완 필요 | 계산 오류 발생 |
| 최종 판정 | Week 12 | 생존/미션/종합 판정 정상 | 일부 팀 재생성 필요 | 최종 판정 생성 불가 |
| Admin QA | `/admin/:roomId` | 확인 필요 항목 없음 또는 경미 | 주의 항목 있으나 조치 가능 | 심각 항목 미해결 |
| 리허설 샘플 | 밸런스 검증 패널 | 정상 | 점검 필요이나 해석 가능 | 샘플 생성/검증 불가 |
| 교육 리포트 | `/report/:roomId` | 화면과 다운로드 정상 | 출력 형식 일부 보완 필요 | 리포트 생성 불가 |
| 데이터 백업 | JSON 백업 | 다운로드 정상 | 수동 복사 병행 | 백업 불가 |

---

## 3. 운영 가능 조건

다음 조건을 모두 충족하면 **운영 가능**으로 봅니다.

- 방 생성, 참가자 입장, 팀 결정, 산출물 저장이 정상이다.
- Week 12 최종 판정이 생성된다.
- Admin QA에서 심각 항목이 없다.
- 리허설 샘플 1개 이상이 정상 생성된다.
- 교육 리포트 다운로드가 가능하다.
- 운영자가 백업 절차를 알고 있다.

---

## 4. 조건부 가능 조건

아래 상황은 **조건부 가능**으로 볼 수 있습니다.

- CI 실행 이력은 없지만 Vercel 배포와 수동 빌드가 성공했다.
- 리허설 샘플 밸런스가 `점검 필요`지만 강사가 해석 가능한 수준이다.
- 일부 문구나 결과 카드의 표현 보완이 필요하지만 핵심 동선은 작동한다.
- 모바일 화면에서 일부 표가 좁지만 참가자 선택 저장에는 문제가 없다.
- 의존성 lock 파일은 아직 없지만 파일럿 중 설치 환경을 고정해 둔다.

조건부 가능일 때는 운영 전 강사가 참가자에게 제한 사항을 미리 설명합니다.

---

## 5. 보류 조건

아래 상황이 하나라도 있으면 파일럿 운영을 보류합니다.

- 방 생성이 되지 않는다.
- 참가자 입장이 되지 않는다.
- 팀 결정 또는 산출물이 저장되지 않는다.
- 결과 계산 또는 최종 판정이 생성되지 않는다.
- 교육 리포트가 생성되지 않는다.
- Admin QA에서 심각 항목이 남아 있다.
- 데이터 백업이 되지 않는다.

---

## 6. 최종 확인 기록 양식

운영자는 파일럿 전 아래 양식으로 기록합니다.

```text
점검일:
점검자:
방 ID:
리허설 샘플:
빌드 확인:
Admin QA 상태:
리허설 밸런스 상태:
교육 리포트 다운로드:
백업 다운로드:
최종 판정: 운영 가능 / 조건부 가능 / 보류
남은 조치:
```

---

## 7. 최종 한 줄 판단

최종 판단은 이렇게 남깁니다.

> 핵심 교육 동선이 작동하고, 결과와 리포트를 통해 디브리핑이 가능하므로 파일럿 운영을 진행한다.

또는

> 핵심 저장/계산/리포트 중 하나가 불안정하므로 원인 수정 후 다시 리허설한다.
