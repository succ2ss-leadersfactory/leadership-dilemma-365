# 파일럿 운영자 패키지

이 문서는 **「리더십 딜레마 365: 위기의 12주」** 파일럿 운영자가 당일 바로 따라갈 수 있도록 정리한 운영 패키지입니다.

---

## 1. 운영 전 준비

### 필수 탭

| 탭 | 경로 | 용도 |
|---|---|---|
| Host | `/host/:roomId` | 진행 제어, 단계 이동, 결과 계산 후 공개 |
| Guide | `/guide/:roomId` | 강사용 최종 운영 가이드, 동선 점검표, 질문 |
| Admin | `/admin/:roomId` | QA 점검, 리허설 샘플, v2 보정, 재계산, 초기화 |
| Report | `/report/:roomId` | 최종 리포트, Markdown 다운로드, 인쇄/PDF 저장 |
| Team | `/team/:roomId/:teamId` | 팀별 입력 화면 |

기본 대면 운영은 **팀당 1개 Team 화면**으로 진행합니다. Player 화면은 온라인 또는 확장 운영에서만 선택적으로 사용합니다.

---

## 2. 파일럿 당일 운영 흐름

### Round 0

1. Host에서 방을 만든다.
2. 팀별 Team 화면을 연다.
3. 각 팀이 KSA 9개를 선택한다.
4. KSA 저장 후 Host에서 완료 팀 수를 확인한다.
5. Week 1을 연다.

### Week 1~11

반복 순서는 다음입니다.

```text
상황 읽기
→ 1분 개인 생각
→ 팀 토의
→ 팀 최종 선택 저장
→ 산출물 저장
→ 결과 계산 후 공개
→ 결과 카드 확인
→ 다음 Week 열기
```

Host 화면에서 결과를 공개하기 전에는 아래 2가지를 확인합니다.

- 팀 최종 선택 저장 완료
- 산출물 저장 완료

### Week 12

1. 개인 성찰은 필요 시 Player 화면 또는 별도 활동지로 수집한다.
2. 팀 선언문은 Team 화면에서 저장한다.
3. 팀 선언문에는 다음이 보이도록 안내한다.
   - 지킬 기준
   - 멈출 습관
   - 이어갈 행동
   - 첫 확인 시점
   - 남는 부담
4. Host에서 최종 판정을 생성한다.
5. Report 화면에서 교육 리포트를 확인한다.

---

## 3. 파일럿 직전 기능 동선 점검

`/guide/:roomId`의 **파일럿 직전 기능 동선 점검표**에서 한 팀 기준으로 P1~P9를 확인합니다.

| 코드 | 점검 동선 |
|---|---|
| P1 | 방 생성과 Host 진입 |
| P2 | 팀 화면과 KSA 저장 |
| P3 | Week 이동과 상황 표시 |
| P4 | 팀 최종 선택 저장 |
| P5 | 산출물 저장 |
| P6 | 결과 계산 후 공개 |
| P7 | Week 12 선언문과 최종 판정 |
| P8 | 교육 리포트 저장 |
| P9 | Admin QA와 초기화 |

P1~P9를 한 팀 기준으로 막힘 없이 통과하면 파일럿 운영을 시작할 수 있습니다.

## 4. 리허설 샘플 4종 사용법

Admin 화면에서 리허설 샘플을 생성할 수 있습니다.

| 샘플 | 목적 |
|---|---|
| 미션 성공형 | 정상 운영, 생존/미션 달성 흐름 확인 |
| 에이스 과부하형 | 빠른 실행과 에이스 소진 부담 디브리핑 |
| 루머 대응형 | 불안, 신뢰, 협업 부담 디브리핑 |
| 재편 위험형 | 낮은 산출물과 반복 속도 선택의 대가 확인 |

샘플 생성 후 확인할 화면은 다음입니다.

1. Admin: 리허설 밸런스 검증
2. Guide: 강사용 질문과 개입 가이드
3. Report: 교육 리포트와 Markdown 다운로드
4. Competencies: 인물 카드와 역량 프로필

---

## 5. 데이터 보정·재계산·초기화

Admin 화면에는 다음 운영 안정화 버튼이 있습니다.

| 버튼 | 사용 시점 |
|---|---|
| JSON 백업 다운로드 | 초기화, 가져오기, 재계산 전 |
| 계산 모델 v2 데이터 보정 | 이전 데이터에 토의 품질, 산출물 품질 v2, 선언문·성찰 피드백이 없을 때 |
| 전체 재계산 + 최종 판정 재생성 | 영향도 TOP 3, 누적 리스크, 최종 게이트가 비어 있거나 오래된 데이터일 때 |
| 현재 방 진행 데이터 초기화 | 리허설 후 실제 운영 전 |
| 전체 DB 초기화하고 처음으로 | 모든 방 데이터를 지우고 새로 시작할 때 |

초기화 전에는 반드시 JSON 백업을 먼저 다운로드합니다.

## 6. 참가자에게 안내할 운영 멘트

운영 시작 시 다음처럼 안내합니다.

> 이 과정은 정답을 맞히는 게임이 아닙니다. 12주 동안 우리 팀이 위기 앞에서 어떤 기준을 반복하고, 그 선택이 어떤 부담을 남기는지 보는 리더십 판단 여정입니다.

결과가 낮게 나온 팀에는 다음처럼 안내합니다.

> 실패팀이라는 뜻이 아닙니다. 현업에서 먼저 낮춰야 할 부담이 드러났다고 보겠습니다.

Week 12에서는 다음처럼 안내합니다.

> 팀 선언문은 멋진 구호가 아니라 다음 회의에서 실제로 확인할 행동 약속입니다.

---

## 7. 참가자에게 노출하지 않을 내부 정보

다음 정보는 강사용 화면과 리포트에는 남기되, 참가자에게 산식처럼 설명하지 않습니다.

- 비밀 미션 세부 기준
- 최종 게이트 내부 기준
- 산출물 품질 계산 비중
- 누적 리스크 원점수
- 인물 카드별 리스크 보정 방식
- 라운드 영향도 TOP 3 계산 기준

설명할 때는 다음 질문으로 바꿉니다.

- 어떤 기준을 반복했습니까?
- 그 선택은 어떤 진전을 만들었습니까?
- 무엇이 남는 부담으로 쌓였습니까?
- 다음 회의에서 무엇을 먼저 바꾸겠습니까?

---

## 8. 종료 후 남길 자료

- Report 화면의 인쇄/PDF 저장본
- Report Markdown 다운로드 파일
- Admin QA 리포트
- JSON 백업
- 강사 회고 메모

파일럿 종료 후에는 새 기능 추가보다, 현장 운영에서 헷갈린 문구와 버튼 흐름을 먼저 수정합니다.
