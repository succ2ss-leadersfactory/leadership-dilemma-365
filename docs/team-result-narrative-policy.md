# 팀별 결과 카드 문장 세분화 정책

이 문서는 같은 선택 유형이라도 팀별 기능 전문성에 따라 결과 해석 문장이 다르게 보이도록 한 기준을 정리합니다.

---

## 1. 목적

기존 결과 카드는 선택 유형과 상태값 중심으로 결과를 보여줬습니다.

이번 보강은 같은 SPEED, STRUCTURE, BALANCE, ALIGN 선택이라도 팀마다 실제 현업 대가가 다르다는 점을 반영합니다.

예를 들어 SPEED 선택은 팀별로 다르게 해석됩니다.

```text
영업팀: 빠른 반응 고객은 잡았지만 기존 고객의 불안이 남을 수 있다.
연구개발팀: 시연 가능한 결과는 만들었지만 기술부채가 남을 수 있다.
HR팀: 빠르게 안내했지만 공정성 논란이 커질 수 있다.
재무팀: 즉각적인 절감 신호는 만들었지만 장기 실행력이 약해질 수 있다.
```

---

## 2. 구현 방식

추가 파일:

```text
src/utils/teamResultNarrativeUtils.js
```

이 파일은 다음 두 기준으로 결과 해석 문장을 반환합니다.

```text
teamId + choiceInternalType
```

선택 유형은 다음 네 가지입니다.

```text
SPEED / STRUCTURE / BALANCE / ALIGN
```

---

## 3. 결과 카드 반영

수정 파일:

```text
src/components/ResultCard.jsx
```

결과 카드에는 다음 섹션이 추가됩니다.

```text
팀별 결과 해석
- 진전
- 남은 대가
- 강사용 질문
```

이 섹션은 기존 결과 카드의 작은 진전, 남은 부담, 조직개편 심사 관점을 대체하지 않고 보완합니다.

---

## 4. 계산 결과 저장

수정 파일:

```text
src/services/calculationService.js
```

라운드 계산 결과에는 `teamResultNarrative`가 저장됩니다.

이를 통해 이후 리포트, QA, 백업 데이터에서도 팀별 결과 해석을 재사용할 수 있습니다.

---

## 5. 설계 원칙

- 결과 카드의 기본 구조는 유지합니다.
- 팀별 문장은 점수 계산을 직접 바꾸지 않습니다.
- 참가자가 바로 이해할 수 있는 직장 언어로 씁니다.
- 강사는 이 문장을 바탕으로 기능 전문성과 리더십 판단을 연결해 디브리핑합니다.
- 같은 선택이라도 각 팀의 합리성과 대가가 다르다는 점을 보여줍니다.

---

## 6. 후속 개선 후보

1. 라운드별 팀 전용 결과 문장 추가
2. 리포트에 팀별 결과 해석 누적 패턴 표시
3. 팀별 result narrative를 비밀 미션 해석과 연결
4. 실제 파일럿 후 어색한 팀별 문장 보정
