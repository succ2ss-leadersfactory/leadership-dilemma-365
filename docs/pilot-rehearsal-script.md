# 파일럿 운영 전 20분 리허설 스크립트

이 문서는 **리더십 딜레마 365: 위기의 12주**를 실제 교육 전에 한 번 빠르게 점검하기 위한 운영 리허설 가이드입니다.

목표는 모든 라운드를 깊게 체험하는 것이 아니라, 강사가 다음 동선이 실제로 연결되는지 확인하는 것입니다.

```text
방 만들기 → 참가자 입장 → KSA 저장 → Week 라운드 진행 → 결과 공개 → Week 12 마무리 → 리포트 저장
```

---

## 리허설 준비

### 준비물

- 운영자 노트북 1대
- 크롬 브라우저 권장
- 참가자 역할 테스트용 브라우저 탭 2~3개
- 필요 시 휴대폰 1대

### 실행

```bash
npm install
npm run dev
```

브라우저에서 Vite 로컬 주소로 접속합니다.

### 리허설 원칙

- 모든 기능을 완벽히 검증하려 하지 않습니다.
- 20분 안에 **핵심 동선이 끊기지 않는지** 확인합니다.
- 실제 교육처럼 길게 설명하지 말고, 화면 이동과 저장 여부를 우선 확인합니다.
- 결과가 낮게 나오는 것은 실패가 아니라 디브리핑 소재입니다.

| 18:00~20:00 | `/report/:roomId` | 리포트 확인 및 Markdown 다운로드 | 파일럿 기록을 저장할 수 있다 |

## 단계별 상세 스크립트

## 1단계. 방 만들기

**화면**

```text
/host/create
```

**강사 멘트**

> 지금부터 실제 교육 전 운영 동선을 한 번 점검하겠습니다. 오늘은 내용을 깊게 토의하기보다, 방 생성부터 리포트 저장까지 끊기지 않는지 확인합니다.

**조작**

1. 기본 콘텐츠 초기화
2. 방 생성
3. 입장 코드 확인

**성공 기준**

- Host Dashboard로 이동한다.
- 입장 코드가 보인다.
- 상단 메뉴에서 Host, Guide, Report, Admin 화면으로 이동할 수 있다.

**문제 시 조치**

- 방이 생성되지 않으면 새로고침 후 다시 시도합니다.
- 기존 localStorage 데이터가 꼬인 경우 새 방을 생성합니다.

---

## 2단계. 참가자 입장

**화면**

```text
/join/:joinCode
```

**강사 멘트**

> 참가자는 먼저 이름과 팀을 입력합니다. 이 게임은 점수 경쟁이 아니라 위기 앞에서 자신의 판단 기준을 확인하는 과정입니다.

**조작**

1. 참가자 탭을 2~3개 엽니다.
2. 이름을 다르게 입력합니다.
3. 서로 다른 팀 또는 같은 팀으로 입장시킵니다.
4. Host 화면에서 참가자 목록을 확인합니다.

**성공 기준**

- Player 화면으로 이동한다.
- Host 화면에 참가자 이름과 팀이 표시된다.
- 참가자 안내 패널이 보인다.

**문제 시 조치**

- 입장 코드가 틀렸다면 Host 화면의 코드를 다시 확인합니다.
- 팀을 잘못 선택한 경우 리허설에서는 새 참가자로 다시 입장합니다.

---

## 3단계. Round 0 KSA 저장

**화면**

```text
/team/:roomId/:teamId
```

**강사 멘트**

> Round 0은 이미 잘하는 역량을 고르는 시간이 아닙니다. 앞으로 12주 동안 팀이 놓치지 않을 기준을 정하는 시간입니다.

**조작**

1. 팀 화면으로 이동합니다.
2. 지식·기술·태도에서 각각 3개씩 선택합니다.
3. KSA 저장 버튼을 누릅니다.
4. `/competencies/:roomId`에서 프로필 생성 여부를 확인합니다.

**성공 기준**

- KSA 9개가 저장된다.
- 팀원별 역량 프로필이 보인다.
- 팀원별 인물 카드가 표시된다.

**문제 시 조치**

- KSA가 저장되지 않으면 각 영역 3개씩 선택했는지 확인합니다.
- 프로필이 보이지 않으면 Admin 화면의 QA 점검표를 확인합니다.

---

## 4단계. Week 1 개인 선택

**화면**

```text
/host/:roomId
/player/:roomId/:playerId
```

**강사 멘트**

> 먼저 혼자 선택합니다. 팀 토의 전에 개인 판단 흔적을 남기는 것이 중요합니다.

**조작**

1. Host 화면에서 Week 1 개인 선택 단계로 이동합니다.
2. Player 화면에서 선택지를 하나 고릅니다.
3. 선택 이유를 짧게 적습니다.
4. 개인 선택 저장을 누릅니다.

**성공 기준**

- 개인 선택이 저장된다.
- Team 화면에서 개인 선택 분포가 보인다.

**문제 시 조치**

- 버튼이 비활성화되어 있으면 Host 화면에서 현재 단계와 화면 잠금 여부를 확인합니다.
- 선택지가 보이지 않으면 현재 라운드가 Week 1인지 확인합니다.

---

## 5단계. 팀 결정과 산출물 저장

**화면**

```text
/team/:roomId/:teamId
```

**강사 멘트**

> 팀 토의에서는 정답을 찾기보다, 이 선택이 만드는 장점과 남기는 부담을 같이 말합니다.

**조작**

1. 팀 최종 선택을 고릅니다.
2. 토론 요약을 한 문장으로 적습니다.
3. 팀 결정 저장을 누릅니다.
4. 산출물 입력 영역에서 핵심 내용을 적습니다.
5. 산출물 저장을 누릅니다.

**성공 기준**

- 팀 결정이 저장된다.
- 산출물이 저장된다.
- 결과 계산 버튼이 활성화된다.

**문제 시 조치**

- 결과 계산 버튼이 비활성화되어 있으면 팀 결정과 산출물이 모두 저장되었는지 확인합니다.
- 산출물은 길게 쓰지 않아도 됩니다. 책임자, 기준, 확인 시점이 보이면 됩니다.

---

## 6단계. 결과 공개

**화면**

```text
/team/:roomId/:teamId
/host/:roomId
```

**강사 멘트**

> 결과는 점수가 아니라 피드백입니다. 작은 진전과 남은 부담을 분리해서 보겠습니다.

**조작**

1. 결과 계산하고 공개를 누릅니다.
2. 결과 카드를 확인합니다.
3. 상태값 변화를 확인합니다.
4. 인물 카드 영향이 표시되는지 확인합니다.

**성공 기준**

- 결과 카드가 보인다.
- 작은 진전과 남은 부담이 보인다.
- 인물 카드 영향 또는 역량 성장 기록이 보인다.

**문제 시 조치**

- 결과 카드가 보이지 않으면 Host 화면에서 계산 후 결과 공개를 다시 실행합니다.
- 계산 오류가 나면 팀 결정과 산출물이 저장되어 있는지 확인합니다.

---

## 7단계. 리허설 샘플 생성

**화면**

```text
/admin/:roomId
```

**강사 멘트**

> 이제 전체 라운드를 빠르게 만든 샘플로 강사 가이드와 리포트가 정상 생성되는지 확인합니다.

**조작**

1. Admin 화면으로 이동합니다.
2. 리허설 샘플 시나리오 중 하나를 선택합니다.
3. 미션 성공형 또는 에이스 과부하형을 먼저 권장합니다.
4. 생성 후 QA 점검표를 확인합니다.

**성공 기준**

- 최종 판정 팀 수가 표시된다.
- 라운드 계산 완료 팀 수가 올라간다.
- 확인 필요 항목이 줄어든다.

**문제 시 조치**

- 생성 후에도 데이터가 보이지 않으면 새로고침합니다.
- 기존 데이터와 섞인 느낌이 있으면 다른 시나리오를 한 번 더 생성합니다.

---

## 8단계. 강사 가이드 확인

**화면**

```text
/guide/:roomId
```

**강사 멘트**

> 강사용 화면에서는 팀의 선택 결과보다, 지금 강사가 어떤 질문을 던질지 확인합니다.

**조작**

1. 강사용 진행 가이드로 이동합니다.
2. 파일럿 진행 체크리스트를 확인합니다.
3. 팀별 관찰·개입 가이드를 확인합니다.
4. 인물 카드 구성과 개입 질문이 보이는지 확인합니다.

**성공 기준**

- 팀별 인물 카드 구성이 표시된다.
- 관찰 신호, 개입 멘트, 질문이 표시된다.
- 현재 라운드에 맞는 강사 멘트가 표시된다.

**문제 시 조치**

- 개입 가이드가 비어 있으면 팀 결정과 결과 계산이 되었는지 확인합니다.
- 샘플 시나리오를 생성한 뒤 다시 확인합니다.

---

## 9단계. 리포트 저장

**화면**

```text
/report/:roomId
```

**강사 멘트**

> 리포트는 교육의 결론이 아니라, 현업에서 무엇을 바꿀지 정리하는 자료입니다.

**조작**

1. 교육 리포트 화면으로 이동합니다.
2. 팀별 최종 판정을 확인합니다.
3. 강사용 관찰 요약을 확인합니다.
4. Markdown 다운로드를 실행합니다.
5. 필요하면 인쇄/PDF 저장을 실행합니다.

**성공 기준**

- 팀별 결과가 보인다.
- 강사용 관찰 요약이 보인다.
- Markdown 파일이 다운로드된다.

**문제 시 조치**

- 최종 판정이 비어 있으면 Admin에서 리허설 샘플을 생성하거나 Team/Host 화면에서 최종 판정을 생성합니다.
- 개인 성찰이 비어 있으면 Player 화면에서 Week 12 성찰을 저장합니다.

9. Admin QA 점검판에서 누락 항목을 확인할 수 있는가?
10. 교육 리포트를 Markdown 또는 PDF로 저장할 수 있는가?

## 파일럿 당일 운영 팁

- 시작 전에 Admin 화면에서 최종 동선 점검표를 한 번 읽습니다.
- 참가자에게 “정답을 맞히는 게임이 아니다”라고 먼저 말합니다.
- 개인 선택을 먼저 저장하게 하고, 팀 토의는 그 다음에 진행합니다.
- 결과가 낮은 팀을 실패팀으로 부르지 않습니다.
- 결과 카드의 숫자보다 “어떤 부담이 남았는가”를 먼저 묻습니다.
- 마지막에는 반드시 개인 성찰과 팀 선언문을 남깁니다.
- 리포트는 교육 후 회고 자료로 저장합니다.
