# 비밀 미션 전문성 증거 기준 정책

이 문서는 팀별 비밀 미션에 산출물 증거 수준 조건을 연결한 기준입니다.

## 목적

비밀 미션은 단순히 좋은 선택을 했는지가 아니라, 팀의 기능 전문성에 맞는 근거를 남겼는지도 보아야 합니다.

따라서 일부 미션 기준은 산출물의 `evidenceReview.evidenceScore`를 확인합니다.

## 새 규칙

새 rule type은 다음입니다.

```text
outputEvidenceAtLeast
```

주요 필드는 다음입니다.

```text
roundIds: 확인할 라운드 목록
minScore: 필요한 증거 점수
label: 미션 근거 문장
```

기본 기준은 `3/4` 이상입니다. 이는 산출물 증거 수준이 최소 `충실` 이상이어야 한다는 뜻입니다.

## 팀별 적용 예

| 팀 | 확인 라운드 | 미션 기준 |
|---|---|---|
| 영업팀 | week2, week11 | 고객 반응 구분과 다음 행동 증거 |
| 마케팅팀 | week2, week11 | 시장 반응, 메시지 기준, 검증 지표 |
| 연구개발팀 | week4, week11 | 기술 기준, 검증 리스크, 협업 조건 |
| 운영팀 | week6, week8, week11 | 병목, 품질 부담, 원팀 공백 보완책 |
| HR팀 | week10, week11 | 공정성 기준, 사실과 추측 구분, 직원 불안 대응 |
| 재무팀 | week4, week11 | 비용 구조, 투자 최소선, 실행 리스크 |

## 판정 방식

지정된 라운드 중 하나라도 `evidenceScore >= minScore`이면 해당 미션 기준을 충족합니다.

예시:

```text
week2 증거 수준 기본(2/4) / week11 증거 수준 충실(3/4) / 기준 3/4 이상
→ 충족
```

## 설계 원칙

- 팀당 비밀 미션 점수는 기존처럼 최대 3점입니다.
- 전문성 증거 기준은 미션 기준 중 일부로만 반영합니다.
- 산출물 글쓰기 점수 경쟁이 되지 않도록 기준은 `충실` 수준으로 둡니다.
- 팀별 기능 전문성과 연결되는 라운드만 확인합니다.

## 후속 점검

- 실제 파일럿에서 미션 달성률이 과도하게 낮아지지 않는지 확인합니다.
- 증거 수준 자동 판정이 실제 산출물과 잘 맞는지 확인합니다.
- 필요하면 `minScore`를 팀별로 조정합니다.
