# 산출물 증거 수준 키워드 보정 정책

이 문서는 산출물 `evidenceReview`가 어떤 표현을 근거로 증거 수준을 판단하는지 정리합니다.

## 목적

파일럿 참가자는 교과서식 용어보다 현업 표현을 씁니다.

따라서 산출물 증거 수준은 단순히 글자 수나 숫자만 보지 않고, 팀 기능 전문성에 맞는 현업 표현도 함께 봅니다.

## 기본 신호

산출물은 다음 신호를 봅니다.

| 신호 | 예시 |
|---|---|
| 판단 근거 길이 | 충분한 설명, 배경, 판단 이유 |
| 구체 증거 | 수치, 빈도, 시점, 금액, 비율, 기간 |
| 다음 행동 | 담당, 기한, 확인, 점검, 합의, 후속 행동 |
| 리스크 신호 | 불안, 부담, 병목, 공백, 지연, 신뢰, 품질, 비용 |
| 팀별 현업 증거 | 고객군, 기술 검증, 병목, 공정성, 비용 구조 등 |

점수는 기존과 동일하게 4점 만점입니다.

팀별 현업 증거가 추가되어도 최대 점수는 4점을 넘지 않습니다.

## 팀별 현업 증거 표현

| 팀 | 감지하는 표현 예시 |
|---|---|
| 영업팀 | 고객군, 핵심 고객, 기존 고객, 반응 구분, 다음 접점, 방문, 팔로우업, 이탈 |
| 마케팅팀 | 타깃, 세그먼트, 시장 반응, 메시지, 포지셔닝, 검증 지표, 캠페인, 전환 |
| 연구개발팀 | 기술 검증, 검증 범위, 기술 리스크, 요구사항, 재작업, 기술부채, MVP, 스펙 |
| 운영팀 | 병목, 납기, 품질, 현장 공백, 예외 기준, 처리 기준, 운영 흐름, 자동화 |
| HR팀 | 공정성, 사실과 추측, 안내 범위, 직원 불안, 루머, 조직개편, 소통 기준 |
| 재무팀 | 비용 구조, 투자 최소선, 실행 리스크, 절감, 예산, ROI, 원가, 손익 |

## 설계 원칙

- 산출물 글쓰기 평가가 되지 않게 합니다.
- 팀별 전문성에 맞는 증거 표현을 더 잘 감지합니다.
- 점수 체계는 기존 4점 만점을 유지합니다.
- 비밀 미션의 `outputEvidenceAtLeast` 기준과 충돌하지 않게 합니다.
- 실제 파일럿 산출물에서 자주 나오는 표현을 계속 보정합니다.

## 후속 점검

파일럿 후에는 다음을 확인합니다.

- 참가자가 많이 쓰는 표현인데 감지되지 않는 단어
- 증거가 부족한데 과하게 높은 점수가 나온 산출물
- 팀별 전문성 표현이 너무 넓거나 좁게 잡히는지
- 비밀 미션 달성률이 지나치게 낮아지는지
