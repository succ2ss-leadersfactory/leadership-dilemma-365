# 팀별 전문지식 렌즈 고도화 계획

이 문서는 리더십 딜레마 365의 사례를 더 깊고 현실적으로 만들기 위한 수정개발 계획입니다.

핵심 방향은 다음입니다.

```text
참가자 화면에는 현실적인 직장 장면을 보여준다.
전문지식과 이론은 선택지, 산출물 기준, 결과 카드, 강사용 해석에 녹인다.
강사는 뒤에서 경영·조직행동·성과관리·직무 전문성으로 디브리핑한다.
```

---

## 1. 1단계: 팀별 전문지식 렌즈 데이터화

추가 파일:

```text
src/data/seedTeamExpertiseLenses.js
```

각 팀에 다음 정보를 부여했습니다.

- businessLens
- leadershipLens
- theoryKeywords
- evidenceStandards
- commonBlindSpots
- facilitatorQuestions

이 데이터는 `gameContent.teamExpertiseLenses`로 포함됩니다.

---

## 2. 2단계: 강사용 가이드 연결

다음 단계에서는 `/guide/:roomId`에 팀별 전문지식 렌즈를 연결합니다.

예시:

```text
영업팀 관찰 포인트:
고객 반응을 관심, 불안, 보류 신호로 구분했는가?

관련 렌즈:
고객가치이해, 시장반응분석, 고객신뢰관리
```

---

## 3. 3단계: 산출물 평가 기준 고도화

팀별 높은 품질 산출물 기준을 다르게 적용합니다.

| 팀 | 높은 품질 산출물 기준 |
|---|---|
| 영업팀 | 고객 반응군, 다음 행동, 신뢰 리스크가 구분됨 |
| 마케팅팀 | 타깃, 메시지, 검증 지표가 연결됨 |
| 연구개발팀 | 기술 가능성, 리스크, 일정 조건이 분명함 |
| 생산/운영팀 | 병목, 책임자, 처리 기준이 명확함 |
| 인사팀 | 공정성, 소통 범위, 불안 관리가 있음 |
| 재무팀 | 비용 구조, 효과, 리스크가 함께 제시됨 |

---

## 4. 4단계: 결과 카드 전문성 신호 추가

결과 카드에 팀별 전문성 부족 신호를 추가합니다.

예시:

```text
전문성 부족 신호:
고객 반응을 숫자로만 보고, 불안의 원인을 구분하지 못했습니다.
```

---

## 5. 5단계: 리포트 전문성 요약 추가

교육 리포트에 팀별 전문지식 렌즈를 반영합니다.

예시:

```text
영업팀은 고객 반응을 빠른 관심 중심으로 해석했지만,
기존 고객의 신뢰 리스크를 별도 행동으로 나누지는 못했습니다.
```

---

## 6. 설계 원칙

- 이론 용어는 참가자 화면에 과도하게 노출하지 않습니다.
- 강사용 화면과 리포트에서는 이론적 해석을 제공합니다.
- 팀별 기능 전문성은 선택의 정답이 아니라 판단 기준으로 사용합니다.
- 기능별 합리성이 충돌하는 장면을 강화합니다.
- 산출물 평가는 글솜씨가 아니라 증거 수준과 판단 구조를 봅니다.
