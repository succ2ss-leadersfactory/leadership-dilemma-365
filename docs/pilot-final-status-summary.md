# v1.0 파일럿 최종 상태 점검 요약표

이 문서는 **리더십 딜레마 365: 위기의 12주** v1.0 파일럿 후보의 현재 준비 상태를 한 장으로 정리한 요약표입니다.

현재 main 브랜치는 파일럿 후보 상태이며, 새 기능 추가보다 최종 리허설과 안정성 확인을 우선합니다.

---

## 1. 완료된 핵심 항목

| 영역 | 상태 | 비고 |
|---|---|---|
| 게임 구조 | 완료 | Round 0, Week 1~11, Week 12 구조 반영 |
| 팀 구조 | 완료 | 6개 팀 구성 |
| 팀별 KSA | 완료 | Round 0 선택 후 역량 프로필 생성 |
| 인물 카드 | 완료 | 팀원 페르소나와 역량 변화 연결 |
| 전체 Week 플레이 | 완료 | Week 1~11 팀 결정과 산출물 구조 반영 |
| 최종 판정 | 완료 | 조직개편 생존, 비밀 미션, 종합 판정 |
| 전문성 렌즈 | 완료 | 6개 팀 전문성 기준 반영 |
| 산출물 증거 수준 | 완료 | evidenceReview 생성 |
| 팀별 결과 해석 | 완료 | teamResultNarrative 반영 |
| 비밀 미션 전문성 기준 | 완료 | outputEvidenceAtLeast 반영 |
| 교육 리포트 | 완료 | 전문성 요약과 누적 경고 신호 포함 |
| Admin QA | 완료 | 리허설 샘플, 전문성 QA, 백업 확인 지원 |
| 운영 문서 | 완료 | 리허설, QA, 인수인계, 문서 패키지 정리 |

---

## 2. 파일럿 전 반드시 확인할 항목

| 항목 | 상태 | 확인 방법 |
|---|---|---|
| npm install | 미확인 | 로컬 또는 CI에서 설치 확인 |
| npm run build | 미확인 | 로컬 빌드 성공 확인 |
| npm run preview | 미확인 | 주요 route 접속 확인 |
| GitHub Actions | 미확인 | main build check 확인 |
| 배포 URL | 미확인 | 최신 main 반영 여부 확인 |
| 리허설 샘플 | 미확인 | Admin에서 최소 2종 생성 |
| JSON 백업 | 미확인 | Admin 백업 다운로드 확인 |
| Report 다운로드 | 미확인 | Markdown 다운로드 확인 |
| 참여자 모바일 | 미확인 | Player/Team 화면 접속 확인 |
| 최종 QA 판정 기록 | 미확인 | 운영 가능/조건부/보류 기록 |

## 3. 파일럿 전 변경 제한

파일럿 전에는 아래 작업을 하지 않습니다.

```text
새 라운드 추가
새 팀 추가
새 상태값 추가
결과 계산 로직 대규모 변경
화면 구조 대폭 변경
의존성 major 업데이트
localStorage key 변경
```

허용되는 변경은 아래로 제한합니다.

```text
오탈자 수정
강사용 문구 보정
문서 보정
QA 체크리스트 보강
명백한 저장/계산/리포트 오류 수정
```

## 6. 최종 판단

현재 버전은 **파일럿 운영 후보**입니다.

기능 구조와 문서 체계는 준비되었으나, 실제 운영 전 아래 확인은 반드시 필요합니다.

```text
npm run build
배포 URL
최종 리허설
백업 다운로드
참여자 모바일 접속
```

위 항목이 통과되면 v1.0 파일럿을 진행할 수 있습니다.
