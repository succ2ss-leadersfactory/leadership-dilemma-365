# v1.0 파일럿 최종 상태 점검 요약표

이 문서는 **리더십 딜레마 365: 위기의 12주** v1.0 파일럿의 현재 준비 상태와 운영 판정을 한 장으로 정리한 요약표입니다.

현재 main 브랜치는 **v1.0 파일럿 운영 가능 상태**로 기록합니다.

---

| 영역 | 상태 | 비고 |

| 게임 구조 | 완료 | Round 0, Week 1~11, Week 12 구조 반영 |
| 팀 구조 | 완료 | 6개 팀 구성 |
| 팀별 KSA | 완료 | Round 0 선택 후 역량 프로필 생성 |
| 인물 카드 | 완료 | 팀원 페르소나와 역량 변화 연결 |

| 최종 판정 | 완료 | 조직개편 생존, 비밀 미션, 종합 판정 |

## 3. 선택 확인 항목

아래 항목은 운영 가능 판정을 뒤집는 조건은 아니지만, 운영 PC 또는 개발자 PC에서 추가로 확인하면 더 안전합니다.

| 항목 | 상태 | 확인 방법 |
|---|---|---|
| npm install | 선택 확인 | 로컬 또는 CI에서 설치 확인 |
| npm run build | 선택 확인 | 로컬 빌드 성공 확인 |
| npm run preview | 선택 확인 | 로컬 미리보기 확인 |
| 참여자 모바일 | 권장 확인 | Player/Team 화면 접속 확인 |
| JSON 백업 파일 보관 | 권장 확인 | Admin 백업 파일 별도 저장 |

## 4. 파일럿 전 변경 제한

파일럿 전에는 아래 작업을 하지 않습니다.

```text
새 라운드 추가
새 팀 추가
새 상태값 추가
결과 계산 로직 대규모 변경
화면 구조 대폭 변경
의존성 major 업데이트
localStorage key 변경
```

허용되는 변경은 아래로 제한합니다.

```text
오탈자 수정
강사용 문구 보정
문서 보정
QA 체크리스트 보강
명백한 저장/계산/리포트 오류 수정
```

## 7. 최종 판단

현재 버전은 **v1.0 파일럿 운영 가능**으로 기록합니다.

다만 파일럿 전에는 새 기능을 추가하지 않고, 아래 운영 안전장치만 유지합니다.

```text
최종 리허설 1회
Admin QA 확인
교육 리포트 다운로드
JSON 백업 다운로드
참가자 모바일 접속 확인
```

위 항목을 운영 당일에도 다시 확인한 뒤 파일럿을 진행합니다.
