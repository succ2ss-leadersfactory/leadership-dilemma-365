# v1.0 파일럿 최종 상태 점검 요약표

이 문서는 **리더십 딜레마 365: 위기의 12주** v1.0 파일럿의 현재 준비 상태와 운영 판정을 한 장으로 정리한 요약표입니다.

현재 main 브랜치는 **v1.0 파일럿 운영 가능 상태**로 기록합니다.

---

## 2. UI/UX 고도화 반영 현황

| 구분 | 상태 | 반영 내용 |
|---|---|---|
| 참가 안내 | 완료 | 첫 접속 시 개인 판단, 이유 기록, 팀 토의, 피드백 확인 흐름 안내 |
| 개인 판단 화면 | 완료 | 판단 기준, 선택 이유 작성 힌트, 개인 판단 저장 문구 보강 |
| 팀 토의 화면 | 완료 | 개인 판단 모아보기, 선택지별 인원 수와 이유 표시 |
| KSA 선택 화면 | 완료 | Round 0 안내, 선택 진행률, 저장 가능 조건 표시 |
| 라운드 상황 카드 | 완료 | 오늘의 판단 질문, 지금 벌어진 일, 팀 신호 구분 |
| 전략 이벤트 카드 | 완료 | 외부 신호, 압박, 놓치기 쉬운 대가, 활용 기회 구분 |
| 12주 타임라인 | 완료 | 현재 위치 표시, compact/전체 모드에 따른 정보 분리 |
| 선택지 카드 | 완료 | 선택 상태, 선택 번호, 판단 힌트가 보이는 카드형 UI |
| 산출물 입력 | 완료 | 실행 약속 중심의 작성 기준과 자연어 피드백 반영 |
| 결과 카드 | 완료 | 작은 진전, 남은 부담, 다음 신호 중심의 피드백 카드화 |
| 최종 판정 카드 | 완료 | Week 12 최종 판정을 마무리 성찰 피드백으로 재구성 |
| 초기 역량 프로필 | 완료 | 교육생용은 강점/주의/성장 초점 중심, 강사용은 상세 표 유지 |

## 3. 교육생 화면 노출 기준

교육생 화면은 아래 기준을 따릅니다.

```text
현재 할 일과 다음 단계는 보인다.
선택의 진전과 부담은 보인다.
팀 토의에 필요한 개인 판단 분포는 보인다.
산출물 작성 기준은 보인다.
결과는 피드백으로 보인다.
강사용 질문, 비밀 미션 점수, 세부 판정 로직, 내부 증거 점수는 과도하게 보이지 않는다.
```

특히 아래 화면은 교육생용으로 정리되었습니다.

```text
PlayerPage
TeamPage
ResultCard
FinalJudgmentCard
StrategicEventCard
TwelveWeekTimeline compact mode
CompetencyProfilePanel 기본 모드
```

## 4. 강사용 화면 유지 기준

교육생 화면에서 숨긴 정보는 강사용 리포트와 상세 모드에서 확인할 수 있어야 합니다.

```text
비밀 미션 제목과 점수
비밀 미션 근거
종합 판정 근거
중간 사건 후폭풍 근거
강사용 질문
인물 카드 영향 기록
팀별 전문성 경고 신호
산출물 증거 수준
역량 프로필 상세 수준 정보
```

현재 `ReportPage`는 강사용 리포트의 기준 화면이며, 전체 타임라인은 자동으로 facilitator 모드로 동작합니다.

## 5. 운영 가능 판정 기록

| 항목 | 상태 | 확인 기준 |
|---|---|---|
| GitHub/Vercel 배포 상태 | 확인 완료 | Vercel status success 확인 |
| 배포 URL 운영 확인 | 사용자 확인 | Vercel 기준 운영 가능 |
| 핵심 기능 구조 | 완료 | 방 생성, 팀 결정, 산출물, 결과 카드, 리포트 구조 반영 |
| 교육생 UX | 완료 | 개인 판단, 팀 토의, 산출물, 결과 피드백 흐름 보강 |
| 강사용 리포트 | 완료 | 판정 근거, 강사용 질문, 전문성 요약 유지 |
| 운영 문서 | 완료 | 최종 리허설, QA 판정, 문서 인덱스, UI/UX QA, 종료 후 회고 템플릿 정리 |
| 최종 운영 판정 | 운영 가능 | 사용자 최종 확인 기준 |

## 6. 선택 확인 항목

아래 항목은 운영 가능 판정을 뒤집는 조건은 아니지만, 운영 PC 또는 개발자 PC에서 추가로 확인하면 더 안전합니다.

| 항목 | 상태 | 확인 방법 |
|---|---|---|
| npm install | 선택 확인 | 로컬 또는 CI에서 설치 확인 |
| npm run build | 선택 확인 | 로컬 빌드 성공 확인 |
| npm run preview | 선택 확인 | 로컬 미리보기 확인 |
| 참여자 모바일 | 권장 확인 | Player/Team 화면 접속 확인 |
| JSON 백업 파일 보관 | 권장 확인 | Admin 백업 파일 별도 저장 |
| UI/UX QA 체크리스트 | 권장 확인 | `docs/pilot-ui-ux-qa-checklist.md` 기준으로 리허설 |
| 종료 후 회고 템플릿 | 권장 확인 | `docs/pilot-post-run-retrospective-template.md` 기준으로 운영 후 기록 |

## 7. 파일럿 전 변경 제한

파일럿 전에는 아래 작업을 하지 않습니다.

```text
새 라운드 추가
새 팀 추가
새 상태값 추가
결과 계산 로직 대규모 변경
화면 구조 대폭 변경
의존성 major 업데이트
localStorage key 변경
```

허용되는 변경은 아래로 제한합니다.

```text
오탈자 수정
강사용 문구 보정
문서 보정
QA 체크리스트 보강
명백한 저장/계산/리포트 오류 수정
교육생 화면의 혼선 제거 수준의 경미한 문구 수정
```

## 10. 최종 판단

현재 버전은 **v1.0 파일럿 운영 가능**으로 기록합니다.

다만 파일럿 전에는 새 기능을 추가하지 않고, 아래 운영 안전장치만 유지합니다.

```text
최종 리허설 1회
Admin QA 확인
UI/UX QA 체크리스트 확인
교육 리포트 다운로드
JSON 백업 다운로드
참가자 모바일 접속 확인
운영 종료 후 회고 템플릿 준비
```

위 항목을 운영 당일에도 다시 확인한 뒤 파일럿을 진행합니다.
