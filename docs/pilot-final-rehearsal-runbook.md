# 파일럿 전 최종 리허설 실행 가이드

이 문서는 실제 파일럿 직전에 운영자가 20~30분 안에 따라 하는 최종 리허설 실행표입니다.

기존 `docs/pilot-rehearsal-script.md`가 기본 동선 확인용이라면, 이 문서는 **최종 운영 가능 여부 판정**까지 연결하는 실행용 문서입니다.

---

## 1. 리허설 목표

최종 리허설의 목표는 기능을 모두 깊게 체험하는 것이 아닙니다.

아래 다섯 가지가 끊기지 않는지 확인합니다.

```text
방 생성
→ 참가자 입장
→ 팀 결정과 산출물 저장
→ 결과 카드와 전문성 신호 확인
→ 최종 판정과 리포트 저장
```

전문성 고도화 기능은 아래 네 가지를 확인합니다.

```text
전문성 렌즈
산출물 evidenceReview
팀별·라운드별 teamResultNarrative
비밀 미션 outputEvidenceAtLeast
```

## 2. 준비물

| 준비물 | 기준 |
|---|---|
| 운영자 노트북 | Chrome 권장 |
| 참가자 테스트 탭 | 2~3개 |
| 모바일 확인 | 필요 시 휴대폰 1대 |
| 기록 문서 | `docs/pilot-final-qa-decision-table.md` |
| 백업 폴더 | QA 리포트, 교육 리포트, JSON 백업 저장 |

## 3. 30분 리허설 실행표

| 시간 | 화면 | 조작 | 성공 기준 |
|---|---|---|---|
| 0~3분 | `/host/create` | 방 생성 | 입장 코드가 보인다 |
| 3~5분 | `/join/:joinCode` | 테스트 참가자 2~3명 입장 | Host에 참가자가 보인다 |
| 5~8분 | `/team/:roomId/:teamId` | Round 0 KSA 9개 선택 | 역량 프로필과 인물 카드가 생성된다 |
| 8~11분 | `/player/:roomId/:playerId` | Week 1 개인 선택 저장 | Team 화면에 선택 분포가 보인다 |
| 11~14분 | `/team/:roomId/:teamId` | 팀 결정과 산출물 저장 | evidenceReview가 생성된다 |
| 14~17분 | Team/Host | 결과 계산과 공개 | 결과 카드가 보인다 |
| 17~20분 | 결과 카드 | 전문성 신호와 팀별 결과 해석 확인 | teamResultNarrative가 표시된다 |
| 20~23분 | `/admin/:roomId` | 리허설 샘플 생성 | QA 수치가 채워진다 |
| 23~25분 | `/admin/:roomId` | 전문성 QA 확인 | 렌즈·증거·결과해석 항목이 보인다 |
| 25~27분 | `/guide/:roomId` | 강사용 렌즈와 질문 확인 | 팀별 전문성 질문이 보인다 |
| 27~29분 | `/report/:roomId` | 전문성 요약과 경고 신호 확인 | 누적 해석이 보인다 |
| 29~30분 | Report/Admin | Markdown/QA/JSON 저장 | 파일이 다운로드된다 |

---

## 4. 전문성 QA 체크

리허설 샘플을 생성한 뒤 Admin과 Report에서 아래를 확인합니다.

| 항목 | 확인 위치 | 정상 기준 |
|---|---|---|
| 팀별 전문성 렌즈 | Guide, Admin | 6개 팀 모두 표시 |
| 산출물 증거 수준 | Team, ResultCard, Admin | evidenceReview 생성 |
| 현업 증거 표현 | ResultCard | 팀 전문성에 맞는 현업 증거 표현 신호 표시 |
| 팀별 결과 해석 | ResultCard, Report | teamResultNarrative 표시 |
| 라운드별 결과 해석 | ResultCard | Week 맥락이 반영된 문장 표시 |
| 누적 경고 신호 | Report | warningSignals 또는 누적 대가 표시 |
| 비밀 미션 전문성 기준 | FinalResult, Report | outputEvidenceAtLeast 충족/미충족 문장 표시 |

## 5. 운영 가능 판정

아래 네 가지가 모두 정상이면 운영 가능으로 봅니다.

```text
핵심 동선 정상
최종 판정 생성
리포트 다운로드 가능
전문성 QA 핵심 항목 정상
```

아래는 조건부 가능입니다.

```text
문구가 일부 어색하지만 저장·계산·리포트가 정상
현업 증거 표현 감지가 일부 부족하지만 evidenceReview는 생성
비밀 미션 전문성 기준 때문에 달성률이 낮지만 강사가 의도를 설명 가능
```

아래는 보류입니다.

```text
방 생성/입장/저장/계산/리포트 중 하나가 작동하지 않음
evidenceReview가 생성되지 않음
teamResultNarrative가 표시되지 않음
outputEvidenceAtLeast 기준이 평가되지 않음
백업 다운로드가 되지 않음
```

최종 판정은 `docs/pilot-final-qa-decision-table.md`에 기록합니다.

## 7. 강사 최종 멘트 점검

리허설 마지막에는 아래 멘트를 실제로 한 번 읽어 봅니다.

> 이 게임은 정답을 맞히는 게임이 아닙니다. 위기 상황에서 우리 팀이 어떤 기준으로 판단하고, 그 선택이 어떤 부담을 남기는지 확인하는 과정입니다.

전문성 신호를 설명할 때는 이렇게 말합니다.

> 산출물 증거 수준은 글을 잘 썼는지 평가하는 점수가 아닙니다. 상무나 타부서가 믿을 수 있는 판단 근거, 다음 행동, 남는 리스크가 남아 있는지를 보는 신호입니다.

결과가 낮은 팀에게는 이렇게 말합니다.

> 이 결과는 실패 판정이 아니라, 현업에서 먼저 다뤄야 할 부담을 보여주는 신호입니다.

## 8. 리허설 후 남길 파일

최종 리허설 후에는 아래 파일을 남깁니다.

```text
교육 리포트 Markdown
Admin QA 리포트 Markdown
JSON 백업 파일
리허설 기록 양식
수정 필요 문구 목록
```

---

## 9. 다음 수정 우선순위 판단

수정 우선순위는 아래 순서로 봅니다.

1. 저장·계산·리포트가 막히는 문제
2. 최종 판정 또는 비밀 미션 기준 오류
3. 전문성 QA가 비어 보이는 문제
4. 참가자가 이해하기 어려운 문구
5. 강사용 디브리핑을 더 풍부하게 만드는 문구

기능 추가보다 파일럿 안정성을 우선합니다.
