# 파일럿 운영 전 UI/UX QA 체크리스트

이 문서는 `리더십 딜레마 365: 위기의 12주` 파일럿 운영 전에 강사와 운영자가 확인할 UI/UX 기준을 정리한 문서입니다.

## 1. QA 원칙

이번 점검의 목적은 기능을 새로 늘리는 것이 아니라, 교육생이 게임 흐름을 자연스럽게 이해하고 강사가 운영 중 혼선을 줄이는 것입니다.

확인 원칙은 다음과 같습니다.

- 교육생 화면에는 정답, 판정 로직, 비밀 미션 세부 점수, 강사용 질문이 과도하게 노출되지 않는다.
- 교육생은 현재 단계에서 무엇을 해야 하는지 바로 이해할 수 있다.
- 강사용 리포트와 전체 타임라인에는 디브리핑에 필요한 상세 정보가 남아 있다.
- route, localStorage key, 계산 로직, 저장 구조는 변경하지 않는다.
- 결과는 평가표가 아니라 다음 판단을 위한 피드백으로 읽힌다.

## 2. 교육생 화면 QA

### Player 화면

확인할 항목:

- 참가 안내가 첫 접속 시 자연스럽게 보인다.
- `혼자 판단 → 이유 기록 → 팀 토의 → 피드백 확인` 흐름이 이해된다.
- 개인 판단 저장 버튼의 의미가 선택 저장보다 판단 기록에 가깝게 보인다.
- 선택 이유 입력란에 판단 근거 작성 힌트가 보인다.
- 팀 화면 이동 링크가 자연스럽게 작동한다.
- 강사용 리포트, 비교 화면, 내부 판정 기준으로 바로 이동하는 요소가 보이지 않는다.

운영 중 관찰 질문:

- 교육생이 “지금 개인 선택을 하는 단계인지, 팀 결정을 하는 단계인지” 헷갈리지 않는가?
- 선택 이유가 단순 감상이 아니라 판단 기준, 얻는 것, 남는 부담을 포함하는가?

### Team 화면

확인할 항목:

- 현재 진행 단계 안내가 보인다.
- 개인 판단 모아보기에서 선택지별 인원 수와 이유가 확인된다.
- 개인 판단 분포가 결론이 아니라 토의 출발점으로 읽힌다.
- 팀 최종 선택 영역에서 팀으로 감수할 선택을 정한다는 안내가 보인다.
- 산출물 입력 영역에서 선택 기준, 다음 행동, 책임자, 확인 시점, 남는 부담을 남기도록 안내된다.
- 결과 공개 전에는 팀 결정과 산출물 저장이 모두 필요하다는 안내가 보인다.
- Host, Report, Compare 직접 이동 버튼이 교육생 화면에 노출되지 않는다.

운영 중 관찰 질문:

- 팀이 다수결로 바로 넘어가지 않고 선택이 갈린 이유를 말하는가?
- 산출물이 긴 글이 아니라 실행 약속으로 작성되는가?

### ResultCard

확인할 항목:

- 결과 카드 상단에 `이번 선택으로 얻은 것`, `아직 남은 부담`, `다음에 먼저 볼 신호`가 보인다.
- 교육생 화면에서는 강사용 질문, 세부 증거 점수, 상세 판정 로직이 보이지 않는다.
- 결과가 승패나 평가가 아니라 다음 라운드 피드백으로 읽힌다.
- 성찰 질문은 짧고 토의에 바로 쓸 수 있다.

운영 중 관찰 질문:

- 교육생이 결과를 점수로만 보지 않고 다음 행동으로 연결하는가?
- 팀이 작은 진전과 남은 부담을 함께 말하는가?

### Week 12 최종 판정

확인할 항목:

- 최종 판정은 순위표가 아니라 마무리 피드백 카드로 보인다.
- 교육생 화면에서는 비밀 미션 점수, 중간 사건 처리 건수, 세부 판정 근거 전체가 과도하게 노출되지 않는다.
- 주요 피드백과 마무리 성찰 질문이 보인다.
- 팀 선언문 작성 흐름과 최종 판정 생성 흐름이 이어진다.

운영 중 관찰 질문:

- 팀이 최종 판정을 결과 확인으로 끝내지 않고 현업 행동으로 연결하는가?
- 팀 선언문이 실제 행동 문장으로 작성되는가?

## 3. 강사용 화면 QA

### Report 화면

확인할 항목:

- 전체 팀 수, 주의 이상 리스크 팀, 생존/조건부 생존 팀, 미션 달성/부분 달성 팀이 요약 카드에 보인다.
- 12주 타임라인 전체 보기에서는 강사용 prompt가 확인된다.
- 강사용 관찰 요약에는 인물 카드 구성, 선택 유형 분포, 관찰 요약, 개입 제안, 핵심 질문이 보인다.
- 팀별 전문성 요약에는 평균 증거 수준, 반복된 진전, 반복된 대가, 경고 신호가 보인다.
- 팀별 결과에는 비밀 미션 점수와 판정 근거가 남아 있다.
- Markdown 다운로드와 인쇄/PDF 저장이 가능하다.

운영 중 활용 질문:

- 어떤 팀이 결과는 냈지만 부담을 크게 남겼는가?
- 어떤 팀이 미션은 달성했지만 조직개편 생존 리스크가 남았는가?
- 어떤 팀이 반복적으로 같은 판단 유형을 선택했는가?

### Facilitator용 정보 유지 확인

교육생 화면에서 숨긴 정보가 강사용 리포트에 남아 있는지 확인합니다.

- 비밀 미션 제목과 점수
- 비밀 미션 근거
- 종합 판정 근거
- 중간 사건 후폭풍 근거
- 강사용 질문
- 인물 카드 영향 기록
- 팀별 전문성 경고 신호
- 산출물 증거 수준

## 4. 공통 컴포넌트 QA

### RoundCard

확인할 항목:

- 오늘의 판단 질문, 지금 벌어진 일, 팀 안에서 보이는 신호가 구분된다.
- 상황 설명이 한 덩어리로 뭉쳐 보이지 않는다.

### StrategicEventCard

확인할 항목:

- 외부 신호, 압박, 놓치기 쉬운 대가, 활용할 수 있는 기회가 구분된다.
- 교육생 화면에서는 강사용 토의 질문이 숨겨진다.
- 강사용 모드에서는 facilitatorPrompt가 유지된다.

### TwelveWeekTimeline

확인할 항목:

- TeamPage의 compact 모드는 교육생용으로 동작한다.
- ReportPage의 전체 모드는 강사용으로 동작한다.
- 현재 위치 배지가 보인다.
- PLAY와 LOG가 구분된다.

### ChoiceList

확인할 항목:

- 선택지가 카드형으로 보인다.
- 선택된 카드는 명확히 표시된다.
- 선택 전 힌트가 보여 선택의 대가를 함께 생각하게 한다.

### CompetencyProfilePanel

확인할 항목:

- 교육생 기본 화면에서는 평균 점수, 초기 대비 변화, 상세 테이블이 숨겨진다.
- 강점 신호, 주의 신호, 성장 초점, 판단 습관이 중심으로 보인다.
- 강사용 모드에서는 상세 수준 정보와 테이블을 확인할 수 있다.

## 5. 파일럿 운영 전 최소 테스트 시나리오

### 시나리오 A: 교육생 1명 개인 판단

1. Player 링크로 입장한다.
2. 참가 안내를 확인한다.
3. 라운드 상황을 읽는다.
4. 선택지를 고른다.
5. 선택 이유를 작성한다.
6. 개인 판단을 저장한다.
7. Team 화면으로 이동한다.

성공 기준:

- 개인 판단이 팀 화면에 반영된다.
- 교육생 화면에서 강사용 리포트 버튼이 보이지 않는다.

### 시나리오 B: 팀 최종 결정

1. Team 화면에서 개인 판단 모아보기를 확인한다.
2. 팀 최종 선택을 저장한다.
3. 산출물을 입력한다.
4. 결과를 공개한다.
5. ResultCard를 확인한다.
6. 다음 라운드로 이동한다.

성공 기준:

- 팀 결정과 산출물 저장 후에만 결과 공개가 가능하다.
- ResultCard가 평가표가 아니라 피드백 카드로 읽힌다.

### 시나리오 C: Week 12 마무리

1. 개인 성찰을 저장한다.
2. 팀 선언문을 저장한다.
3. 최종 판정을 생성한다.
4. FinalJudgmentCard를 확인한다.
5. Report 화면에서 전체 리포트를 확인한다.

성공 기준:

- 교육생 화면에서는 세부 판정 로직이 과도하게 보이지 않는다.
- Report 화면에서는 강사용 판정 근거와 질문이 확인된다.

## 6. 남은 점검 후보

다음 항목은 파일럿 후 교육생 반응을 보고 추가 개선 여부를 결정합니다.

- 모바일 화면에서 카드 간 간격과 버튼 위치 조정
- ReportPage에 강사용 상세 컴포넌트 재사용 확대
- FacilitatorGuidePage와 ReportPage의 역할 분리 강화
- 팀별 최종 결과를 PDF용 요약 카드로 별도 출력
- 산출물 입력 품질에 따른 예시 문장 추가
