# 파일럿 운영자 1페이지 런북

이 문서는 `리더십 딜레마 365: 위기의 12주` 파일럿 당일에 강사와 운영자가 순서대로 확인할 진행표입니다.

## 1. 운영 전 10분 확인

| 확인 항목 | 기준 |
|---|---|
| 배포 URL 접속 | 교육생용 Player/Team 화면 접속 가능 |
| 강사용 화면 접속 | Report/Admin/운영 화면 접속 가능 |
| 방 생성 | 파일럿용 room 생성 또는 기존 room 확인 |
| 참가자 링크 | 개인별 Player 링크와 팀별 Team 링크 준비 |
| 백업 | 운영 전 JSON 백업 1회 저장 |
| 브라우저 | 강사용 PC와 예비 PC에서 접속 확인 |
| 모바일/태블릿 | 교육생 대표 기기 1대 이상 접속 확인 |

운영 전 멘트:

```text
오늘 게임은 정답을 맞히는 활동이 아니라, 위기 상황에서 각자가 어떤 기준으로 판단하고 팀이 어떤 대가를 감수하는지 확인하는 과정입니다.
```

## 2. 시작 안내

교육생에게 먼저 알려줄 것:

```text
1. 먼저 혼자 판단합니다.
2. 왜 그렇게 판단했는지 짧게 남깁니다.
3. 팀 화면에서 서로의 판단 차이를 확인합니다.
4. 팀으로 감수할 선택을 정합니다.
5. 산출물에는 다음 행동과 남는 부담을 남깁니다.
6. 결과 카드는 점수가 아니라 다음 판단을 위한 피드백입니다.
```

강사가 피해야 할 말:

```text
정답을 맞혀 보세요.
점수가 높은 선택을 고르세요.
이 선택이 제일 좋습니다.
비밀 미션 기준을 미리 알려드리겠습니다.
```

## 3. Round 0 KSA 선택 진행

진행 순서:

1. 팀별 Team 화면으로 이동합니다.
2. 지식, 기술, 태도에서 각 3개씩 선택하게 합니다.
3. KSA 저장 후 초기 역량 프로필이 생성되는지 확인합니다.
4. Week 1 개인 선택 단계로 이동합니다.

강사용 관찰 포인트:

- 팀이 강점만 고르는가, 실제 필요한 기준을 고르는가?
- 지식, 기술, 태도 중 특정 영역에만 토의가 쏠리는가?
- KSA 선택이 이후 팀 결정의 기준으로 다시 언급되는가?

## 4. Week 1~11 기본 진행 루프

각 라운드는 아래 순서로 운영합니다.

```text
상황 읽기
↓
개인 판단 저장
↓
팀 화면에서 개인 판단 모아보기
↓
팀 최종 선택 저장
↓
산출물 입력
↓
결과 공개
↓
결과 카드 토의
↓
다음 라운드 이동
```

### 개인 판단 단계

강사 멘트 예시:

```text
지금은 팀 토의 전에 각자 먼저 판단하는 시간입니다. 다른 사람 의견을 보기 전에 내가 본 핵심 문제와 선택 이유를 남겨 주세요.
```

확인 기준:

- 모든 참가자가 개인 선택을 저장했는가?
- 선택 이유가 비어 있지 않은가?
- 선택 이유에 핵심 문제, 얻는 것, 남는 부담 중 하나 이상이 들어갔는가?

### 팀 토의 단계

강사 멘트 예시:

```text
이제 다수 의견을 바로 고르지 말고, 왜 의견이 갈렸는지 먼저 이야기해 주세요. 팀으로 감수할 선택을 정하는 것이 목표입니다.
```

확인 기준:

- Team 화면의 개인 판단 모아보기가 보이는가?
- 팀이 최종 선택을 저장했는가?
- 토론 요약이 단순 결론이 아니라 선택 기준을 포함하는가?

### 산출물 단계

강사 멘트 예시:

```text
산출물은 긴 글쓰기 과제가 아닙니다. 회의가 끝난 뒤 바로 실행할 행동, 책임자, 확인 시점, 남는 부담을 남기는 기록입니다.
```

확인 기준:

- 필수 입력란이 비어 있지 않은가?
- 다음 행동이 구체적인가?
- 책임자와 확인 시점이 보이는가?
- 남는 부담을 숨기지 않았는가?

### 결과 공개 단계

강사 멘트 예시:

```text
결과 카드는 평가표가 아니라 피드백입니다. 이번 선택으로 얻은 것과 아직 남은 부담을 확인하고 다음 라운드에서 무엇을 줄일지 이야기해 주세요.
```

확인 기준:

- ResultCard가 정상 생성되는가?
- 작은 진전, 남은 부담, 다음 신호가 보이는가?
- 강사용 내부 정보가 교육생 화면에 과도하게 노출되지 않는가?

## 5. Week 12 마무리 진행

진행 순서:

```text
개인 성찰 저장
↓
팀 선언문 작성
↓
최종 판정 생성
↓
최종 판정 카드 확인
↓
교육 리포트 확인
↓
디브리핑
```

개인 성찰 멘트:

```text
이번에는 내가 12주 동안 반복한 판단 습관과 현업에서 바로 바꿀 행동을 적어 주세요.
```

팀 선언문 멘트:

```text
우리 팀이 위기 상황에서 지키려 했던 기준과 현업으로 가져갈 행동을 한 문장으로 정리해 주세요.
```

최종 판정 멘트:

```text
최종 판정은 순위표가 아닙니다. 조직개편 압박 속에서 우리 팀이 무엇을 지켰고 무엇을 뒤로 미뤘는지 확인하는 마무리 피드백입니다.
```

## 6. 강사용 리포트 확인

Report 화면에서 확인할 항목:

- 전체 팀 수
- 주의 이상 리스크 팀
- 조직개편 생존/조건부 생존 팀
- 미션 달성/부분 달성 팀
- 중간 사건 후폭풍
- 대표 판단 패턴
- 개인 성찰 제출 수
- 강사용 관찰 요약
- 팀별 전문성 요약
- 팀별 최종 판정 근거

리포트 저장:

```text
1. 인쇄 / PDF 저장
2. Markdown 다운로드
3. JSON 백업 다운로드
```

## 7. 디브리핑 질문 세트

공통 질문:

1. 우리 팀은 위기 앞에서 속도, 기준, 균형, 조건 중 무엇을 반복했습니까?
2. 그 판단은 어떤 성과를 만들었고, 어떤 부담을 남겼습니까?
3. 개인 판단과 팀 최종 선택은 어디에서 달라졌습니까?
4. 산출물에 남긴 다음 행동은 실제 현업에서 실행 가능한 수준이었습니까?
5. 조직개편 생존 판정과 미션 달성 판정이 다르게 나온 팀은 왜 그렇게 되었습니까?
6. 다음 현업 회의에서 바로 바꿀 행동 하나는 무엇입니까?

팀별 추가 질문:

- 빠른 실행형 선택이 많았던 팀: 무엇을 빨리 얻었고, 누구에게 부담이 몰렸습니까?
- 기준 정리형 선택이 많았던 팀: 기준은 명확해졌지만 실행 속도는 늦어지지 않았습니까?
- 균형 조정형 선택이 많았던 팀: 갈등은 줄었지만 어려운 결정을 미루지는 않았습니까?
- 조건 조율형 선택이 많았던 팀: 위와 옆을 맞추는 동안 현장 실행이 느려지지는 않았습니까?

## 8. 운영 중 문제 발생 시 대응

| 상황 | 대응 |
|---|---|
| 참가자가 Player 화면을 잃어버림 | 개인 링크를 다시 전달 |
| 팀원이 개인 판단을 저장하지 않음 | Team 화면에서 누락 여부 확인 후 개인 저장 요청 |
| 팀 결정 저장 전 결과 공개 불가 | 팀 최종 선택과 산출물 저장 여부 확인 |
| 산출물이 비어 있음 | 다음 행동, 책임자, 확인 시점, 남는 부담 중 최소 2개 이상 작성 요청 |
| 결과 카드가 보이지 않음 | 결과 공개 버튼 실행 여부 확인 |
| 최종 판정이 보이지 않음 | Week 12에서 최종 판정 생성 여부 확인 |
| 리포트 저장 필요 | Report 화면에서 PDF 저장, Markdown 다운로드, JSON 백업 진행 |

## 9. 운영 종료 후 기록

운영 종료 후 강사는 아래를 별도로 기록합니다.

```text
가장 몰입이 높았던 라운드
가장 혼선이 컸던 화면
교육생이 가장 많이 질문한 부분
팀 토의가 가장 깊어진 질문
산출물 작성에서 어려웠던 항목
다음 파일럿 전 반드시 고칠 점
```

## 10. 최종 운영 원칙

파일럿 당일에는 새 기능을 추가하지 않습니다.

허용되는 조정은 아래로 제한합니다.

```text
오탈자 수정
강사용 안내 멘트 보정
문서 보완
명백한 저장 오류 대응
교육생 화면 혼선 제거 수준의 경미한 문구 수정
```

운영의 핵심은 화면을 완벽하게 보여주는 것이 아니라, 교육생이 매 라운드마다 아래 질문으로 돌아오게 하는 것입니다.

```text
우리는 무엇을 얻었고, 무엇을 뒤로 미뤘는가?
다음 현업에서는 무엇을 다르게 할 것인가?
```
